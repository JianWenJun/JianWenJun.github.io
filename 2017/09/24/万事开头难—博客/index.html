万事开头难—博客

发表于 2017-09-24 | 分类于 程序人生 | | Views:
| 字数统计: 1,243 | 阅读时长 ≈ 4 分钟

开头难

计划写些技术博客很久了，然而每次都是计划的很好，实施的过程却因为这事或者那事给耽搁了:cry:，其实这都是借口，归咎原因还是自己过于墨迹。但是每每想到自己之前学到的东西或者解决的问题在过一段时间就忘记了什么也没有留下，很是痛苦，觉得这次必须把写博客这件事坚持下来。==没有借口，永不放弃！！！==

谈原因

坚持写技术博客的原因有很多，知乎等平台上到处可以查找到，但是在这我想结合自己的切身体会，总结出几点自己认为重要的原因。

  • ==作为学习笔记==，都说好记性不如烂笔头，可事实其实如此，特别是作为程序员，要学的东西特别多，==编程语言的语法，程序的算法思路，程序的设计技巧（设计模式），项目的架构==都需要去学习，这些技术性的东西可能当时学起来快，但是忘起来也快。这时候，很有必要记下一些自己当时学习的笔记，有助于自己后来的快速回顾。

    不知道大家有没有出现过这种情况，与别人交谈或者看别人的博客的时候，提及到一些关键词或 者一些关于技术名词的简称，感觉很熟悉但是怎么也想不起是什么意思。回去后自己又急忙的去 搜去百度。这时候，我觉得一开始如果能将这些东西有意识的作为学习笔记模块记录下来，回去 的时候可能直接便能找出来，而且自己做的笔记更容易加深理解。

  • ==作为问题集==，作为一名程序员，多多少少会遇到一些技术上的问题，==要么是程序代码上的问题，要么是开发软件上的问题，要么是算法思路上的问题==。相信当时的这些问题可能会卡住你不止一时半会，让你很是痛苦，可是当你解决了的时候，又是多么的兴奋。你遇到的问题可能也是别人的问题，相信这时候应该能体会到他的痛苦。把自己的问题整理下来，一方面可以帮助别人，一方面有利于自己回顾再次遇到时候能很快的解决。

  • ==作为技术分享==，其实搞计算机程序的人还是很乐意分享自己的技术积累的，共同进步嘛，毕竟以后到公司了还是会有技术分享大会的。将自己会的技术分享出来，不仅可以帮助正在学习这门技术的人学习，如果有幸被大神看到，说不定还能被人家指点一二，利于自身的提高。对于以后找工作其实也是很有帮助的。

    谈规划

    说了这么多，无非是想激励自己能把写技术博客这件事给坚持下去。但是还是不能停留在口头上，这次得把规划给写下来，先做起来，行动起来!!!

  • 经历
    对于一个刚步入广州某学校的计算机相关学院的研究僧，算是一个萌新了，暂时可以不用管找工作了还是继续读研的问题了，可以一心搞技术了或者说搞研究了。大学四年稍纵即逝，回顾一下有得又有失吧。说一下自己的技术经历吧，也是Java出身，大学的四年里大部分时间都花在移动开发这个方向上了，目前为止，能独立的开发==Android==和==iOS==的APP了，后端自己能写些但是很烂，主要的话还是使用一些第三方BaaS服务来作为后端开发APP。大学时光也不算啥也没学吧。

  • 计划

    博客既然已经开篇了，就得有自己的博客计划了。

    1.==以博客的形式整理大学时积累的技术==，由于本人很懒，本科学的技术和遇到的问题积累了一些却没有整理很是混乱，借着研究僧这段算平静的时光，整理好技术文档。以备自己回顾的同时或许能帮助到他人（大神请忽略）。
    2.==以博客的形式记录研究僧时期遇到的技术问题和学习笔记==，相信接下来的技术学习阶段一定更精彩，自己要学习的东西更多（计划学习的方向：机器学习的算法，自然语言处理，软件架构—-==路漫漫其修远兮，吾将上下而求索==），这下得有意识的去积累去整理。

==没有借口，永不放弃！！！==
